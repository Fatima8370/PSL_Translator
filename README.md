# PSL Translator

A real-time Pakistani Sign Language (PSL) translator — built with Python, OpenCV, and MediaPipe — that turns hand signs into text, with support for regional dialect variation.

## What it does

Using nothing but a standard webcam, this project detects hand landmarks in real time, normalizes them into a dialect-agnostic feature vector, and classifies them into the corresponding letter or number using a trained machine learning model. The long-term goal is a full desktop application and browser extension that can translate PSL — alphabets, numbers, and eventually full words and phrases — into text, accounting for the fact that PSL is signed differently across different regions and communities in Pakistan.

## Why this project exists

I was on a road trip, and stopped at a fast food chain for a break from the driving. While waiting in line, I watched an employee at the counter trying to sign something to the customer in front of him. It was clear neither of them really knew what to do — not because anyone was unwilling, but because almost no one around them had any exposure to sign language at all.

That moment stuck with me. Sign language awareness in Pakistan is minimal — there are very few public programs for the Deaf and disabled community, and PSL isn't even taught in most schools. The result is a communication gap that shows up in small, everyday moments like that one, all across the country, for people who have no easy way to bridge it themselves.

This project won't fix that gap on its own, but it's a step toward making everyday communication a little more possible — starting with something as simple as a camera and a translated letter on a screen.

## Current status: Phase 1

- [x] Real-time hand landmark detection (MediaPipe Tasks API, `HandLandmarker`, VIDEO mode)
- [x] Dialect-agnostic, translation- and scale-invariant landmark normalization
- [x] Two-handed sign support with a fixed-length feature vector
- [x] Timed-burst dataset collector, organized by dialect for easy multi-style data gathering
- [x] Random Forest classifier trained on collected landmark data
- [x] Live prediction overlay on the webcam feed
- [ ] Full A–Z, 0–9 dataset across 2 dialects
- [ ] Word/phrase-level (motion-based) sign support

- [ ] Desktop application

## Tech stack

- **Python** — primary language
- **OpenCV** — webcam capture, rendering
- **MediaPipe (Tasks API)** — hand landmark detection
- **scikit-learn** — landmark-based classification
- **NumPy / pandas** — feature processing and dataset handling

## Project structure

```

├── config.py                 # all tunable constants (paths, thresholds, etc.)
├── landmarker_service.py      # wraps MediaPipe HandLandmarker
├── feature_extraction.py      # raw landmarks -> normalized feature vector
├── drawing.py                 # all OpenCV overlay/rendering logic
├── prediction_service.py      # loads trained model, runs inference
├── main.py                    # live webcam + prediction entry point
├── data_collector.py          # timed-burst tool for recording training data
├── load_dataset.py            # combines all dialect CSVs into one dataset
├── train_model.py             # trains and saves the classifier
├── dataset/
│   ├── dialect_1/
│   │   ├── A.csv
│   │   └── B.csv
│   └── dialect_2/
│       └── ...

```

## Notes on design

- **Dialect-blind labels:** signs from different dialects (e.g. `dialect_1/B.csv` and `dialect_2/B.csv`) are treated as different training examples of the *same* letter, not different classes. The goal is one letter in, one letter out — regardless of which signing style was used.
- **Landmark-based, not pixel-based:** classification runs on normalized hand-landmark coordinates rather than raw images, keeping the dataset small, training fast, and the model robust to lighting, hand size, and position in frame.
